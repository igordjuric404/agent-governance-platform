# Rogue Agent Detection — Business-Oriented Description

## Executive Summary

**Rogue Agent Detection** is Ophanix's continuous security monitoring system for AI agents. It uses behavioral analysis to automatically detect when an agent has become compromised, malfunctioning, or dangerous — and takes action before damage is done.

## The Problem

AI agents can go rogue through:

| Threat Vector | How Agent Goes Rogue |
|--------------|---------------------|
| Prompt Injection | Malicious inputs manipulate agent behavior |
| Compromise | Attacker gains control of agent |
| Goal Drift | Agent gradually偏离s intended purpose |
| Bug/Malfunction | Coding errors cause dangerous behavior |
| Resource Exhaustion | System issues cause erratic behavior |

**Without detection:** You might not know until damage is done.

## Value Proposition

| Without Detection | With Rogue Agent Detection |
|-----------------|--------------------------|
| Unknown until incident | Immediate detection |
| Manual monitoring | Automated surveillance |

| Incident unknown until customer complaint | Proactive alerting |

## How It Works

1. **Establish Baseline** — Learn what "normal" looks like for each agent
2. **Monitor Continuously** — Watch every action for deviations
3. **Detect Anomalies** — Flag unusual behavior patterns
4. **Assess Risk** — Calculate how dangerous the anomalies are
5. **Take Action** — Kill switch, alert, or continue monitoring

## Key Features

- **Per-Agent Baselines** — Learns individual normal behavior
- **Multi-Vector Detection** — Action patterns, resource access, timing, velocity
- **ML-Powered** — Continuously improving detection
- **Automatic Response** — Kill switch for critical threats
- **Low False Positives** — Per-agent learning reduces alert fatigue
